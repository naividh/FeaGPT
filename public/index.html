FeaGPT: End-to-End Agentic AI for FEA

FeaGPT is a natural-language-driven framework that automates the complete Finite Element Analysis workflow: from engineering description through geometry creation, mesh generation, simulation, and results analysis.

GMSA Pipeline Architecture

The core pipeline follows four stages (Geometry-Mesh-Simulation-Analysis), each orchestrated by an LLM agent with domain-specific knowledge retrieval.

Pipeline: NL Description → Planning → Geometry (G) → Meshing (M) → Simulation (S) → Analysis (A)

Key Equations

Knowledge retrieval uses cosine similarity with threshold σ > 0.85:

sim(q, d) = cos(θ) = (q · d) / (||q|| · ||d||)

Fatigue life prediction uses Basquin's equation:

σa = σ'f (2Nf)b

Adaptive batch sizing (Eq. 7):

Bsize = min(Mavail / Mper_case, Ncores, Bmax)

Mesh quality metric (Eq. 8):

Q = Jdet / Jideal, where AR = max(edge) / min(edge) ≤ 10

Supported Materials

Built-in material database includes Al-7075-T6, Al-6061-T6, Steel-AISI-4340, Ti-6Al-4V with full S-N curve data for fatigue analysis.

API Reference

The REST API provides endpoints for programmatic access:

POST /analyze   POST /batch   GET /status/{id}   GET /results/{id}   GET /health

Technology Stack

Python backend with FastAPI, CalculiX solver, Gmsh meshing, FreeCAD geometry. Frontend served via Firebase Hosting. CI/CD through GitHub Actions.
